## 📋 剩余步骤（需手动完成）

### 1️⃣ 创建 Cloudflare Worker

1. 访问 https://workers.cloudflare.com
2. 注册/登录账号
3. 创建 Worker，命名为 `vote-api`
4. 粘贴 `worker.js` 代码
5. 点击 Deploy

### 2️⃣ 创建 KV 存储

1. Workers Dashboard → KV
2. Create a namespace → 命名 `vote-data`
3. 复制 Namespace ID

### 3️⃣ 绑定 KV 到 Worker

1. Worker → Settings → Variables
2. Add binding:
   - Variable: `VOTE_KV`
   - Namespace: `vote-data`
3. Save and Deploy

### 4️⃣ 获取 Worker URL

部署后你会获得类似这样的 URL：
```
https://vote-api.你的账号.workers.dev
```

### 5️⃣ 更新前端配置

编辑 `index.html` 第 156 行：
```javascript
const API_BASE = 'https://vote-api.你的账号.workers.dev';
```

### 6️⃣ 部署前端到 GitHub Pages

```bash
cd ~/.openclaw/workspace/projects/vote
git push -u origin main
```

然后在 GitHub 仓库设置中启用 Pages。

## 💡 为什么需要后端？

**原问题**：使用 localStorage，数据只在本地浏览器
**解决方案**：Cloudflare Workers + KV 云端存储

**优势**：
- ✅ 所有用户数据实时同步
- ✅ 免费额度足够 500 人投票
- ✅ 无需自己维护服务器
